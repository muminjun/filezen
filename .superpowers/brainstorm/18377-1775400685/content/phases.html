3 페이즈 로드맵

이 순서로 구현하는 걸 추천합니다. 각 페이즈가 독립적으로 동작하는 소프트웨어를 만들어요.

1

Phase 1 — 핵심 완성 + 빠른 UX 개선 (A, B, G, H, J)

A 실제 이미지 변환 연결  ·  B 처리 진행률 + 전/후 비교  ·  G 라이트박스  ·  H 썸네일 크기 슬라이더  ·  J 키보드 단축키

앱의 핵심 가치 완성. 이것만 해도 "쓸 수 있는 이미지 변환기"가 됩니다.

2

Phase 2 — 파워 유저 편의 (I, K, E)

I 정렬 + 포맷 필터 + 검색  ·  K 드래그로 순서 변경  ·  E 일괄 파일명 변경 패턴

이미지가 많을수록 빛나는 기능들. Phase 1이 완성된 뒤에 추가하면 자연스러워요.

3

Phase 3 — 고급 편집 (C, D, F)

C 이미지 크롭  ·  D 밝기/대비/채도 조절  ·  F 워터마크

구현 난이도가 가장 높고, 앞의 두 페이즈가 탄탄해야 제값을 해요.

각 페이즈별로 스펙 → 계획 → 구현을 독립적으로 진행합니다.
Phase 1 스펙부터 바로 작성할게요. 동의하면 "ㅇㅇ" 해주세요.
