# filezen 기능 확장 로드맵 설계 문서

**작성일**: 2026-04-12  
**담당자**: minjun  
**상태**: 확정

---

## 개요

filezen은 현재 이미지 변환·편집(이미지 탭)과 PDF 처리(파일 탭)를 제공하는 클라이언트사이드 파일 툴킷이다. 이 문서는 서비스 전체 사용자(개인·크리에이터·비즈니스)를 커버하기 위한 18개 신규 기능의 설계를 정의한다.

모든 기능은 **서버 비용 없이 클라이언트사이드**로 구현하는 것을 원칙으로 한다.

---

## Phase 1 — 이미지 탭 강화

### 1. 이미지 리사이즈

**목적**: 현재 크롭만 가능한 이미지 탭에 실제 해상도 변경 기능 추가

**기능 상세**:
- 입력 단위: px (절대값) 또는 % (비율)
- 가로/세로 비율 잠금 옵션
- 여러 장 선택 후 일괄 적용
- 리사이즈 결과는 ZIP 다운로드 시 반영

**구현 방식**: Canvas API — `canvas.width/height` 설정 후 `drawImage()` 리샘플링

**UI 위치**: EditDrawer 내 새 섹션 또는 BottomActionBar에 버튼 추가

---

### 2. 워터마크 삽입

**목적**: 저작권 보호 및 브랜딩을 위한 워터마크 일괄 삽입

**기능 상세**:
- 타입: 텍스트 워터마크 / 이미지 워터마크
- 텍스트: 폰트·크기·색상·불투명도 설정
- 위치: 9개 앵커 포인트 (좌상·중앙·우하 등) + 자유 위치
- 반복 패턴 모드 (타일 형태로 전체 반복)
- 선택된 이미지 전체 일괄 적용

**구현 방식**: Canvas API — `ctx.drawImage()` + `ctx.fillText()` 합성

---

### 3. AI 배경 제거

**목적**: 인물·사물 이미지의 배경을 자동 제거하여 PNG 투명 배경으로 출력

**기능 상세**:
- 클라이언트사이드 AI 추론 (서버 없음)
- 처리 후 결과 미리보기 제공
- 일괄 처리 지원 (여러 장 순차 처리)
- 출력 포맷: PNG (투명도 유지)

**구현 방식**: `@imgly/background-removal` 라이브러리 (WebAssembly 기반)

**주의사항**: 최초 실행 시 모델 다운로드 (~40MB), 로딩 프로그레스 UI 필요

---

### 4. EXIF 메타데이터 제거

**목적**: 이미지에 포함된 GPS 위치·촬영 기기 정보 등 개인정보 일괄 제거

**기능 상세**:
- 제거 대상: GPS 좌표, 촬영 날짜, 카메라 기종, 작성자 정보 등
- 선택된 이미지 일괄 적용
- 처리 후 원본 파일명 유지

**구현 방식**: Canvas API를 통한 재인코딩 — `canvas.toBlob()` 시 EXIF가 자동 제거됨 (별도 라이브러리 불필요)

---

## Phase 2 — PDF 탭 완성

### 5. PDF 비밀번호 보호

**목적**: 현재 잠금 해제(UnlockTool)만 있는 PDF 탭에 비밀번호 설정 기능 추가

**기능 상세**:
- 사용자 비밀번호(열기용) + 소유자 비밀번호(편집 제한용) 별도 설정
- 권한 설정: 인쇄·복사·편집 허용 여부
- 처리 후 즉시 다운로드

**구현 방식**: `pdf-lib` 라이브러리의 암호화 API

---

### 6. PDF 워터마크

**목적**: PDF 문서 전체 페이지에 텍스트/이미지 워터마크 삽입

**기능 상세**:
- 텍스트 워터마크: 폰트·크기·색상·불투명도·각도 설정
- 대각선 반복 패턴 지원
- 전체 페이지 일괄 적용
- 특정 페이지 범위 선택 가능

**구현 방식**: `pdf-lib` — 각 페이지에 콘텐츠 스트림 추가

---

### 7. 전자서명 (드로우 서명)

**목적**: 마우스/터치로 서명을 그려 PDF에 삽입하는 간단한 전자서명

**기능 상세**:
- 서명 캔버스: 마우스·터치 드로우 지원
- 서명 이미지 저장 (재사용 가능)
- PDF 내 삽입 위치 선택 (페이지·좌표)
- 크기 조정 가능

**구현 방식**: Canvas API로 서명 이미지 생성 → `pdf-lib`로 PDF에 이미지 삽입

---

### 8. PDF → 이미지 추출

**목적**: PDF 각 페이지를 이미지 파일로 추출

**기능 상세**:
- 출력 포맷: PNG / JPEG 선택
- 해상도(DPI) 설정: 72 / 150 / 300
- 전체 페이지 또는 특정 페이지 범위 선택
- ZIP으로 일괄 다운로드

**구현 방식**: 기존 `pdfThumbnail.ts` 로직 확장 — `pdf.js` 렌더링 + Canvas → Blob

---

## Phase 3 — 신규 카테고리

### 9. 동영상 → GIF / 이미지 변환

**목적**: 동영상 파일을 GIF 또는 이미지 프레임으로 변환하는 신규 카테고리

**기능 상세**:
- 입력 포맷: MP4, MOV, WebM
- 출력: GIF 변환 (시작·끝 시간, FPS, 크기 설정) 또는 프레임별 이미지 추출
- 처리 진행률 표시 (용량이 크므로 프로그레스 필수)

**구현 방식**: `ffmpeg.wasm` (클라이언트사이드 FFmpeg)

**주의사항**: 번들 크기 증가 (~30MB) — 동적 임포트(lazy load) 필수

---

### 10. 아이콘 / 파비콘 생성기

**목적**: 이미지 1장으로 모든 플랫폼 아이콘 사이즈를 자동 생성

**기능 상세**:
- 출력 세트:
  - 파비콘: `favicon.ico` (16/32/48px), `favicon-32x32.png`, `favicon-16x16.png`
  - Apple: `apple-touch-icon.png` (180x180)
  - Android/PWA: 192x192, 512x512
  - OG Image: 1200x630
  - Windows Tile: 150x150
- 전체 세트 ZIP 다운로드
- 배경색 설정 (투명 또는 단색)

**구현 방식**: Canvas API 리사이즈 + `ico` 포맷은 직접 바이너리 인코딩

---

### 11. 소셜 미디어 사이즈 프리셋

**목적**: 플랫폼별 권장 이미지 사이즈로 원클릭 변환

**기능 상세**:
- 지원 플랫폼·규격:
  - Instagram: 피드 정사각형(1080×1080), 피드 세로(1080×1350), 스토리(1080×1920), 릴스(1080×1920)
  - Twitter/X: 포스트(1600×900), 프로필(400×400), 헤더(1500×500)
  - YouTube: 썸네일(1280×720), 채널 아트(2560×1440)
  - LinkedIn: 포스트(1200×628), 커버(1584×396)
- 크롭 방식: center-crop (자동) 또는 letter-box (여백 추가) 선택
- 여러 장 선택 시 일괄 적용 후 ZIP 다운로드

---

### 12. 프리폼 콜라주 에디터

**목적**: 레이아웃을 자유롭게 설계하고 여러 사진을 1장으로 병합

**기능 상세**:

**캔버스 조작**:
- 셀 분할: 셀 선택 후 가로/세로 분할 버튼으로 자유 분할
- 디바이더 드래그: 셀 간 구분선을 드래그하여 비율 실시간 조절
- 셀 병합: 인접 셀 병합 기능
- 전체 비율: 1:1 / 4:5 / 9:16 / 16:9 / 자유

**사진 배치**:
- 셀 클릭 또는 드래그로 사진 배치
- 셀 간 드래그로 사진 위치 교체
- 셀 내 핀치줌·드래그로 사진 크기·위치 조정 (object-fit: cover 방식)

**스타일**:
- 셀 간격(gap): 0px ~ 20px
- 외곽 여백(padding)
- 배경색 / 그라데이션
- 셀 모서리 라운드

**출력**: 전체 캔버스를 1장으로 병합 → PNG/JPEG/WebP 다운로드, 해상도 선택

**구현 방식**: CSS Grid 기반 에디터 UI + 최종 출력 시 Canvas API로 래스터화

---

### 13. 이미지 OCR (텍스트 추출)

**목적**: 이미지 속 텍스트를 인식하여 복사 가능한 텍스트로 추출

**기능 상세**:
- 언어 선택: 한국어, 영어, 자동 감지
- 인식 결과 텍스트 표시 + 클립보드 복사
- 인식 영역 시각화 (바운딩 박스 오버레이)

**구현 방식**: `Tesseract.js` (WebAssembly 기반, 클라이언트사이드)

**주의사항**: 언어 데이터 최초 다운로드 (~10MB/언어) — 로딩 UI 필요

---

### 14. 색상 팔레트 추출

**목적**: 이미지에서 주요 색상을 추출하여 디자인 작업에 활용

**기능 상세**:
- 주요 색상 5~10개 추출
- HEX / RGB / HSL 형식으로 표시
- 색상 클릭 시 클립보드 복사
- 전체 팔레트 JSON/CSS 변수 형식으로 복사

**구현 방식**: Canvas API `getImageData()` + k-means 색상 군집화 알고리즘 (직접 구현 또는 `color-thief` 라이브러리)

## Phase 4 — UX 개선

### 15. 클립보드 붙여넣기

**목적**: Ctrl+V / ⌘+V로 이미지를 즉시 추가하는 편의 기능

**기능 상세**:
- 전역 키보드 이벤트 리스너 (`paste` 이벤트)
- 클립보드 이미지 → 현재 활성 탭에 자동 추가
- 스크린샷 직접 붙여넣기 지원

**구현 방식**: `navigator.clipboard.read()` + `ClipboardEvent.clipboardData`

---

### 16. 전체화면 드래그 앤 드롭존

**목적**: 페이지 어디서든 파일을 드롭하면 자동으로 올바른 탭에 추가

**기능 상세**:
- 브라우저 전체 영역에 `dragover` 이벤트 감지
- 파일 타입 자동 분류: 이미지 → 이미지 탭, PDF → 파일 탭
- 드롭 중 시각적 오버레이 표시

---

### 17. 작업 히스토리 / 실행 취소

**목적**: 편집 작업을 단계별로 되돌릴 수 있는 Ctrl+Z 지원

**기능 상세**:
- 이미지 탭: 회전·뒤집기·색보정·크롭·리사이즈 등 편집 히스토리 저장
- Ctrl+Z (실행 취소) / Ctrl+Shift+Z (재실행) 단축키
- 히스토리 최대 20단계

**구현 방식**: AppContext에 `history: ImageFile[][]` 스택 추가, 액션마다 스냅샷 저장

---

### 18. URL로 이미지 가져오기

**목적**: 이미지 URL을 입력하여 업로드 없이 바로 추가

**기능 상세**:
- URL 입력 → 이미지 fetch → Blob으로 변환 후 추가
- CORS 이슈 시 프록시 경유 or 안내 메시지

**구현 방식**: `fetch(url)` → `response.blob()` → 기존 `addImages()` 파이프라인

## 기술 의존성 요약

| 기능 | 라이브러리 | 번들 크기 |
|------|-----------|---------|
| AI 배경 제거 | `@imgly/background-removal` | ~40MB (모델, lazy) |
| PDF 보호·워터마크·서명 | `pdf-lib` | ~800KB |
| 동영상 변환 | `ffmpeg.wasm` | ~30MB (lazy) |
| OCR | `Tesseract.js` | ~10MB/언어 (lazy) |
| 색상 팔레트 | `color-thief` 또는 직접 구현 | ~10KB |
| 아이콘·리사이즈·워터마크 | Canvas API (내장) | 0 |

> 모든 대용량 라이브러리는 동적 임포트(`import()`)로 필요 시점에만 로드

---

## 네비게이션 구조 변경

현재 이미지 탭 / 파일 탭 2개 구조에서 Phase 3 이후 신규 카테고리 추가 시:

```
이미지 탭 | 파일(PDF) 탭 | 소셜 스튜디오 탭 | 변환 탭
```

- **소셜 스튜디오 탭**: 소셜 미디어 프리셋 + 프리폼 콜라주 에디터
- **변환 탭**: 동영상→GIF, 아이콘 생성기, OCR, 색상 팔레트

---

## 개발 순서 원칙

1. Phase 1 → 2 → 3 → 4 순차 진행
2. 각 Phase 내에서 난이도 낮은 기능 먼저 (EXIF 제거 → 리사이즈 → 워터마크 → AI 배경제거 순)
3. 대용량 라이브러리(ffmpeg.wasm, Tesseract)는 Phase 3 마지막에 배치
4. Phase 4 UX 개선은 전체 개발 중 언제든 병행 가능
